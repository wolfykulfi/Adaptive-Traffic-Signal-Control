## Project Overview

**Adaptive Traffic Signal Control (ATSC)** using Deep Q-Network (DQN) reinforcement learning with SUMO traffic simulation.

The AI agent learns to control traffic signal phases at a 4-way intersection to minimize vehicle waiting time and congestion, outperforming traditional fixed-timing signals.

## Common Commands

Always activate the virtual environment first and use `-X utf8` flag on Windows to avoid encoding errors:

```bash
# Activate venv (run in PowerShell terminal, not here)
venv\Scripts\activate

# Train the model (500 episodes, ~30 minutes)
python -X utf8 scripts/train.py

# Evaluate trained model vs fixed-timing (opens SUMO-GUI)
python -X utf8 scripts/evaluate.py

# Generate all result plots
python -X utf8 scripts/plot_results.py

# Run SUMO-GUI manually to inspect intersection
sumo-gui --configuration-file sumo_configs/intersection.sumocfg

# Regenerate SUMO network after editing nod/edg/con files
netconvert \
  --node-files=sumo_configs/intersection.nod.xml \
  --edge-files=sumo_configs/intersection.edg.xml \
  --connection-files=sumo_configs/intersection.con.xml \
  --output-file=sumo_configs/intersection.net.xml \
  --tls.default-type=actuated --no-turnarounds true
```

## RL Design

| Component | Detail |
|-----------|--------|
| **State** | 32-dim vector: 16 queue lengths + 16 waiting times (all normalised 0–1) |
| **Actions** | 4 discrete signal phases: NS straight (0), NS left (2), EW straight (4), EW left (6) |
| **Reward** | `-total_halting_vehicles / (MAX_QUEUE × N_LANES)` → range [-1, 0] |
| **Decision step** | Every 10 simulation seconds |
| **Min green time** | 10 seconds before a phase change is allowed |
| **Yellow duration** | 4 seconds (inserted automatically on phase change) |

### DQN Hyperparameters (configs/train_config.py)
| Parameter | Value | Notes |
|-----------|-------|-------|
| `lr` | 5e-4 | Lower than default to prevent Q-value divergence |
| `gamma` | 0.99 | Long-term reward focus |
| `epsilon_start` / `epsilon_end` | 1.0 → 0.05 | |
| `epsilon_decay` | 0.998 | Slow decay — more exploration |
| `buffer_capacity` | 50,000 | Large buffer for stable sampling |
| `batch_size` | 128 | |
| `target_update` | 50 | Infrequent target sync = more stable |
| `n_episodes` | 500 | |
| `max_steps` | 3600 | 1 simulated hour per episode |
| `save_every` | 50 | Intermediate checkpoints saved as `models/dqn_ep{N}.pth` |

## SUMO Intersection

- **Type**: 4-way signalised, single junction (node ID: `C`)
- **Arms**: North, South, East, West — each 500m long
- **Lanes**: 4 per direction (8-lane road total per arm)
- **Lane layout** (right → left in direction of travel):
  - Lane 0: right turn
  - Lane 1: straight
  - Lane 2: straight
  - Lane 3: left turn
- **TLS phases** (generated by netconvert):
  - Index 0: NS straight green
  - Index 1: NS straight yellow
  - Index 2: NS left green
  - Index 3: NS left yellow
  - Index 4: EW straight green
  - Index 5: EW straight yellow
  - Index 6: EW left green
  - Index 7: EW left yellow
- **Traffic**: Probabilistic flows — cars, trucks (West only), motorcycles. Varied density per approach (West heaviest, East lightest).

## Key Implementation Notes

- **Always use `python -X utf8`** when running scripts on Windows — Unicode characters in docstrings cause `cp1252` encoding errors otherwise.
- **TraCI connection for evaluation**: `traci.start()` with `label=` works for DQN (sumo-gui); use headless `sumo` for the fixed-timing comparison sim to avoid GUI startup timing issues.
- **Phase indices** in `sumo_env.py` match exactly what netconvert generated — do not renumber without regenerating the network.
- **Reward scale**: The reward is bounded [-1, 0]. Previous attempt used delta-waiting-time which produced values ~0.001, causing loss explosion. Current halting-vehicle reward is stable.
- **Loss function**: SmoothL1Loss (Huber) — robust to Q-value outliers. Do not revert to MSELoss.
- **Model checkpoint**: `models/dqn_final.pth` is the trained model. Load with `agent.load(path)` then set `agent.epsilon = 0.0` for evaluation.
